@Override
Colocado sobre os métodos que irão sobrescrever os métodos de suas superclasses.
Ele não é necessário de ser usado para fazer um override, porém é uma boa prática, caso ocorra uma mudança de nome do método (Ex. ToString → tostring), o qual não permitirá associar o método com o seus correspondente o qual fará o override, fazendo o compilador gerar um erro caso o método não esteja fazendo nenhum override.

@SuppressWarnings
A anotação @SuppressWarnings faz o compilador suprimir avisos para um determinado método.
Por exemplo, se um método chamar um método obsoleto, ou faz uma conversão do tipo inseguro, o compilador pode gerar um aviso.
Você pode suprimir esses avisos anotando o método que contém o código com a anotação @SuppressWarnings.
@SuppressWarnings
public void metodoComAviso() {
	...
}
