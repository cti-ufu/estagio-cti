Porém, a ordem dos catches devem ser feito de menor para maior ... deixando as menores subclasses no topo, e aumentando nos próximos catches.

Se deixar um maior sobre um menor, o código não compilara.

Por exemplo, se for fazer um catch(Exception ex) (Exception é como se fosse a classe Objeto no caso de exceções, ele é A superclasse maior de todas), o compilador compilara apenas o Catch do Exception, não vendo necessidade de fazer o catch de outras exceções menores.
