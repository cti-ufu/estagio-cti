Cap.11 - Comportamento Arriscado

É possível ter riscos quando for chamar métodos de uma classe que você não escreveu. Tal método pode fazer algo arriscado, que acabe impedindo que seus códigos funcionem. Então deve ser escrito um código que possa cuidar da falha, caso ela ocorra.

Para saber qual o tipo de erro que pode ocorrer com tal método, tem que pesquisar no API do Java sobre o método que deseja usar, e procurar em sua página a cláusula Throw, que é onde fala o tipo de erro que pode ocorrer e quando ele ocorre.
